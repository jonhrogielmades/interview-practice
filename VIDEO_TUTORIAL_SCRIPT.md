# InterviewPilot Whole System User Script

Target length: 8 to 10 minutes
Audience: First-time users, evaluators, capstone panelists, and system testers
Goal: Explain what users need before using InterviewPilot, then show the complete user flow from account access to interview practice, feedback, review, and progress tracking.

## Recording Checklist

- Use Chrome or Edge on desktop for the clearest recording.
- Open the app at `http://127.0.0.1:8000` or the configured local URL.
- Prepare one regular user account with a few saved practice sessions.
- Prepare one admin account only if the admin overview will be shown.
- Allow microphone permission if demonstrating voice or hybrid answers.
- Allow camera permission if demonstrating camera readiness or non-verbal coaching.
- Use one clear sample goal, such as "Scholarship interview for an IT student".
- Keep browser zoom at 90% or 100% so the sidebar and workspace fit on screen.
- If showing mobile access, prepare the LAN URL before recording.

## Main Message

"InterviewPilot is an AI-based interview practice system that helps users prepare for job, scholarship, college admission, and IT interviews. A user needs an account, a browser, a chosen interview goal, and optional microphone or camera access. After that, the system guides the user through setup, mock interview practice, AI feedback, session review, progress tracking, and continued learning."

## What Users Need

Before using the system, a user needs:

- An InterviewPilot account or Google sign-in access, if Google authentication is configured.
- A modern browser such as Chrome or Edge.
- A stable internet or local network connection.
- A clear practice goal, such as job interview, scholarship interview, college admission interview, or IT / programming interview.
- A preferred answer mode: text, voice, or hybrid.
- Microphone permission for voice answers and speech-to-text.
- Camera permission for camera readiness and selected non-verbal coaching.
- A few minutes of uninterrupted time for each practice session.

## 20 Advanced Feature Set

These features can be highlighted during the tutorial if they are visible in the current build, included in the prototype, or presented as planned system enhancements:

1. **Interview Anxiety Calibration Mode** - Starts with easier questions and gradually increases pressure as the user improves.
2. **Stress Interview Simulation** - Simulates interruptions, silence, repeated follow-up questions, and time pressure.
3. **Answer Evidence Checker** - Detects vague answers and asks the user for proof, examples, or measurable results.
4. **STAR/CAR Answer Structure Coach** - Checks whether answers follow a proper interview structure such as STAR or CAR.
5. **Resume-Consistency Detector** - Compares spoken answers with the uploaded resume and flags possible contradictions.
6. **Panel Interview Mode** - Uses multiple AI interviewers with different perspectives such as HR, technical, and manager.
7. **Adaptive Weakness Recovery Mode** - If the user performs poorly in one topic, the system automatically gives recovery questions and mini-practice.
8. **Personal Story Bank Builder** - Stores the user's experiences, achievements, projects, and leadership stories for reuse in interviews.
9. **Behavioral Pattern Memory Across Sessions** - Tracks repeated habits such as filler words, weak openings, poor examples, or unfinished answers over time.
10. **Company Culture Match Practice** - Adjusts the interview style depending on startup, corporate, government, or academic settings.
11. **Localized Interview Mode** - Supports country-specific or region-specific interview styles, language tone, and expected etiquette.
12. **Multilingual / Code-Switching Interview Practice** - Lets users answer in English, Filipino, or mixed language for more realistic practice.
13. **Honesty and Overclaim Warning** - Flags answers that sound exaggerated, unsupported, or too generic.
14. **Real-Time Hint Suppression and Fair Mode** - Allows a strict mode with no hints and a guided mode with hints so users can compare both performances.
15. **Post-Interview Reflection Journal** - After every session, asks the user what they felt, where they struggled, and what they learned.
16. **Micro-Lesson Generator** - Converts weak areas into short lessons such as "How to answer Tell me about yourself" or "How to explain weaknesses."
17. **Interview Persona Switching** - Lets users practice with a friendly interviewer, strict recruiter, skeptical manager, or fast-paced technical lead.
18. **Career-Stage Aware Feedback** - Gives different feedback for senior high students, fresh graduates, interns, career shifters, and experienced applicants.
19. **Mock-to-Real Preparation Workflow** - After practice, generates likely final questions, last-minute reminders, and a personal review checklist.
20. **Actionable Improvement Roadmap** - Instead of only giving scores, creates a step-by-step improvement plan for the next 7 or 14 days.

## Short Version For A 2-Minute Demo

1. Open the homepage and introduce InterviewPilot as an AI-based interview practice system.
2. Explain what the user needs: account, browser, interview goal, optional microphone, and optional camera permission.
3. Sign in with a prepared demo account.
4. Open Session Setup and choose category, question count, focus mode, pacing, and response mode.
5. Open Practice, answer one question, and submit.
6. Show AI feedback for clarity, relevance, grammar, professionalism, strengths, improvements, and next step.
7. Save the session, then open Progress or Session Review.
8. Show Chatbot or Learning Lab as continued support.
9. Mention the optional advanced coaching feature set, Profile, Notifications, Mobile LAN, and the optional Admin workspace.
10. Close by saying the system supports repeated interview practice and progress tracking.
